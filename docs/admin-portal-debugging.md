# Admin Portal Chat Debugging

This project’s Admin Chat UI and backend now support lightweight tracing so it’s easier to see **where** a failure happened (browser vs server vs Foundry).

## UI debug mode

Open the dashboard with the `debug` query parameter:

- `http://localhost:3000/dashboard.html?debug=1`

When enabled:

- The chat UI shows a **Debug** panel with a rolling log of request/response events.
- Each agent response shows a `requestId` next to **Agent** for quick correlation.

## Request correlation (`requestId`)

Every `/admin/chat` call includes an `x-request-id` header (generated by the browser). The server:

- logs the `requestId` on start/end/error
- returns the same `requestId` in JSON responses

This makes it easy to match a UI error to server logs.

## Where to look when something fails

### 1) Browser-side

- Open DevTools Console.
- If using `?debug=1`, check the Debug panel at the bottom of the Admin Chat card.

### 2) Server-side

Admin portal logs are written to:

- `admin-portal/logs/server.log`

Search for the request ID shown in the UI, e.g.:

- `[<requestId>] Admin chat ...`
- `[<requestId>] Admin chat success ...`
- `[<requestId>] Admin chat error ...`

### 3) “Empty response” vs “Failure”

The server distinguishes between:

- **Empty agent response**: request succeeded but no text was extractable from the Foundry response. The server returns `{ success: true, empty: true }`.
- **Failure**: request failed due to network/server errors. The server returns `{ success: false }` with HTTP 4xx/5xx.

If you see frequent **empty** responses, it usually indicates the text extraction logic needs to be updated to match the current Foundry response schema.

## MCP tool approvals (manual)

Some prompts trigger MCP tool calls (e.g., calling your backend tools). These can require explicit approval.

- This project does **not** auto-approve MCP tool calls.
- When approval is required, the admin chat API returns HTTP `409` and an error message like:
	- `MCP tool call requires approval in Foundry portal. Pending approval request(s): mcpr_...`

Approve the pending request in the Foundry portal, then retry the same prompt in the Admin Chat UI.
